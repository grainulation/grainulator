“Should we use Postgres or MongoDB for the new service?”
Recommendation
Postgres with JSONB columns for flexible fields. Covers relational queries (80% of workload) without sacrificing document flexibility.
Timeline: 2 weeks (Postgres) vs 9 weeks (MongoDB) 91% confidence
Key finding: 80% of queries are relational joins — MongoDB incurs a significant performance penalty. pgvector eliminates the need for a separate vector database.
high MongoDB migration effort is 4.5× longer than initially estimated
med Ecosystem tooling coverage at 73% — needs evaluation
low Schema flexibility gap mitigated by JSONB columns
